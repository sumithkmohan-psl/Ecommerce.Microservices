# Dockerized .NET Backend Demo with CI/CD Deployment

## Project Overview

This project is a backend-focused ASP.NET Core demo application built to gain practical exposure in modern deployment workflows beyond traditional IIS-hosted enterprise environments.

The application demonstrates how a .NET backend service can be:

* containerized using Docker,
* integrated with an automated CI/CD deployment pipeline,
* and deployed to a cloud hosting platform for environment-independent release management.

The primary objective of this project was to strengthen hands-on familiarity with modern backend engineering practices such as container portability, deployment automation, and cloud-based application hosting.

## Project Objectives

This demo was created as a self-learning engineering exercise to explore:

* how backend applications can be packaged consistently using Docker,
* how code pushes can trigger automated deployment workflows,
* how cloud platforms can simplify release management,
* and how modern backend systems can be made portable outside local server dependencies.

---

## CI/CD Workflow

The project includes a CI/CD-enabled deployment process where:

1. Source code is pushed to GitHub repository
2. Automated build pipeline is triggered
3. Docker image is prepared for deployment
4. Application is deployed to Render cloud hosting environment

This helps simulate a simplified real-world release pipeline.

---

## Docker Containerization

Docker has been used to package the application with all runtime dependencies, ensuring:

* consistent execution across environments,
* simplified deployment,
* and easier portability of backend services.

---

## Cloud Deployment

The containerized application is deployed to Render to understand:

* cloud-based hosting workflows,
* deployment automation,
* and managed runtime execution of backend services.
